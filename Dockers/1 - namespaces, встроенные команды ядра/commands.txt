1)
chroot [options] newrootname [command]  - запускает команду с подменой рута на другой путь
whereis [комманда] - чтобы найти папку с утилитой
ldd [путь к утилите] - библиотеки, используемые утилитой
копируем библиотеки и исполняемые файлы в новый рут, сохраняя пути, чтобы всё работало.

2) 
ip netns - работа с пространством имен для изоляции сетевых интерфейсов
ip netns add newNameSpace - новое пространство имен-контейнер 
ip netns ls - список существующих
ip netns exec newNameSpace bash - добавляем текущий процесс(баш в данном случае) к пространству имен

sudo ip link add veth0 type veth peer name veth1 - создание двух виртуальных интерфейсов, состыкованных друг с другом(link)
делается в другой сессии баш

sudo ip addr add 10.0.0.1/24 dev veth0  - присваиваем ip-адрес на виртуальный интерфейс
sudo ip link set dev veth0 up - включаем его (статус down меняется на up в таблице)
делаем аналогично для другого виртуального интерфейса(ip-address другой)

sudo ip netns del testNetns  - удаляем namespace, а с ним и связанные вирт. интерфейсы

3)
$$ - PID текущего терминала
/proc/N - виртуальная файловая система для каждого отдельного процесса( N = PID), со своими служебными папками
unshare -h  - запускает процесс с изоляцией в собственных namespace по указанным в аргументах областям
ls -l /proc/$$/ns
sudo unshare --net /bin/bash
sudo unshare --pid --net --fork --mount-proc /bin/bash
